How to configure and test

1. Local testing

Keep mock mode. It needs no server or credentials. Run scan os-info in the terminal.

2. Real management server

Select http and enter the server API URL. This local test server uses http://127.0.0.1:8080; every non-loopback deployment must use HTTPS, for example https://manager.company.local.

3. Credentials

You do not paste an auth token here. On Start/Restart, registration receives the agent ID, auth token, and encryption key from the server and stores them in the OS keychain or restricted identity file.

4. Network authorization

Add only CIDRs you own or are authorized to test, one per line, such as 192.168.1.0/24. Then Apply & Restart before running scan network-scan --target 192.168.1.10 --ports 22,80,443.

5. Dependency scanning

Add project directories under SCA paths, one per line, such as C:\Projects. The agent never scans the whole disk by default.
